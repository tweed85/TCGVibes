# PandaBananasTCG

A browser-based Pokémon TCG clone. Vite + React + TypeScript. Plays a human
vs. a strategic AI, local hot-seat style, using the real Play! Pokémon
Standard-format card pool (North America, snapshot 2026-04-23).

## Run

```bash
npm install
npm run dev        # http://localhost:5173
npm run typecheck  # tsc -b --noEmit
npm run test       # vitest
npm run build
```

Node 18+ required.

## Architecture notes

- **Card schema mirrors the Pokémon TCG API.** `cardMapper.ts` is the only
  translation layer; the rest of the engine uses engine types.
- **Effects are data-driven.** Attack text is regex-matched on first use
  (`effectPatterns.ts`) and cached as a structured `AttackEffect[]` on the
  Attack. The runtime resolver (`effects.ts`) dispatches on `kind`.
  Unmatched text is preserved as free-form `text` for display.
- **Trainer and ability effects follow the same pattern.** `detectTrainerEffect`
  and `detectAbilityEffect` return a narrow union; unknown effects don't fire
  but the card still discards / displays.
- **RNG lives on `GameState`.** All coin flips and shuffles go through
  `state.rng` so games with the same seed are reproducible.
- **Lazy dataset load.** `cards.ts` uses dynamic `import()` so the 1.5MB
  JSON ships as its own chunk and parses on first deck-builder open
  rather than at app boot.
- **1-ply lookahead AI.** `ai.ts` clones state via `cloneStateForSearch`
  (JSON-clone with rng/log stripped), simulates each candidate attack +
  opp's greedy response + our follow-up, picks the move that leaves us in
  the best scored position. The `lookaheadActive` flag prevents infinite
  recursion when nested simulations re-enter `tryAttack`.
- **Wild Growth–aware cost checks.** `energyPoolForCost(p, state)` in
  `ongoingEffects.ts` wraps `energyProvidedBy` with ability bonuses (e.g.
  Meganium's Wild Growth doubles each Basic Grass). All attack/retreat
  cost checks route through it.
- **Memoized React renders + stale-closure pattern.** `CardView` is wrapped
  in `React.memo` with a custom comparator that excludes `onClick`. Click
  handlers compensate by reading current state through `stateRef.current`
  instead of relying on captured closure values, so preserved memo
  closures still see fresh game state.
- **Interactive picker pattern.** `PendingInPlayTarget` carries an action
  discriminated union that drives multi-step pickers — Adrena-Brain /
  Cursed Blast (damage-counter movement), Aura Jab (energy-attach
  to Bench), Oil Salvo / Phantom Dive (distributed damage). Resolver
  in `trainerEffects.ts:resolveInPlayTarget` re-arms `pendingInPlayTarget`
  between clicks until `remaining` hits 0.
- **Turn flow handles paused states.** When an Active is KO'd from an
  attack/checkup, the engine sets `pendingPromote` + `phase = "promoteActive"`
  and queues what to run after the promote via `onPromoteResolved`
  ("endTurn" / "passTurn" / "secondAttack"). Ability-induced KOs are
  non-terminal (phase stays `main`) so the player keeps the turn.

## Rules implemented

- 60-card deck, max 4 per name (except basic energy)
- 7-card opening hand, mulligan for no-Basic + **opponent draws N extra**
- 6 Prizes; draw 1 per KO. **ex/V/VSTAR/GX = 2 Prizes; VMAX/V-UNION /
  Mega Evolution ex = 3 Prizes** (`prizeValue` enforces by subtype).
- 5-slot bench; active + bench zones
- 1 Energy attach / turn, 1 Supporter / turn, 1 retreat / turn
- **First player cannot attack or play a Supporter on T1**
- **First-turn evolve gate applies to BOTH players' first turn.**
  `isPlayersFirstTurn(state, player)` returns true on engine T1 for the
  going-first player and engine T2 for the going-second player. Drives
  the evolve check, Rare Candy, and Grimsley's Move.
- Weakness ×2, Resistance −N
- All 5 Special Conditions (Asleep, Burned, Confused, Paralyzed, Poisoned)
  with proper Pokémon Checkup processing + attack-time confusion flip
- **Status immunity** — Festival Grounds (any energy attached →
  status-immune), Insomnia (no asleep), Antique Fossils (all conditions
  blocked).
- Retreat + evolution clear special conditions
- Stadium zone (new Stadium replaces old, discards it)
- Pokémon Tool attach mechanic (max 1 per Pokémon; discarded with holder)
- **Ability-disabling auras.** Initialization, Midnight Fluttering, Sticky
  Bind suppress targeted abilities via `abilitiesActiveOnInstance`.
- **Passive attack-modifying abilities are evaluated** in damage and
  the AI's estimator (`passiveAttackBonus`).
- Bench KOs from snipe / recoil / status / counter-placement resolve with
  prize draws.
- Player-picks-Active on KO (AI auto-picks via `scorePromoteCandidate`
  with 1-ply opp-threat lookahead and counter-OHKO bonus).
- Win conditions: prizes=0, no Pokémon left, opponent can't draw.

## Effect coverage across the 2,693-card pool

**Attack effects: ~70 kinds wired** — coin-flip variants, per-energy /
per-bench / per-counter scaling, status, heal, snipe, multi-target,
self-cant-attack, defender-cant-retreat, retreat / discard manipulation,
draw, and bespoke effects. Highlights:
- `distributeDamage` (Oil Salvo, Phantom Dive bench-only) — interactive
  hit-by-hit picker for humans, prize-greedy auto-distribute for AI.
- `placeCountersPerHandCard` (Alakazam Powerful Hand) — counter
  placement that bypasses W/R and is reflected in the damage preview.
- `useAttackFromOppDeckTop` (Persian ex Haughty Order),
  `useBenchedAllyNamedAttack` (Night Joker),
  `discardTopOfOwnDeckUseSupporterEffect` (Ninetales Supernatural
  Shapeshifter) — copy-attack pipelines.
- `discardDefenderEndOfOppNextTurn` (TR Grimer Corrosive Sludge)
  scheduled-KO via `scheduledKoOnTurn` flag, settled in `endTurn`.
- `attachNFromDiscardToBench` (Aura Jab) — interactive picker for humans.
- `winGameIfPrizesEquals`, `bothActiveKnockedOut`,
  `countersOnEachWithAbility` — bespoke endgame effects.

**Abilities: ~70 activated kinds wired** plus triggered-on-evolve,
triggered-on-bench, triggered-on-move-to-active. Highlights:
- `attachEnergyFromHandThenDraw` (Teal Dance — top score for AI),

  `attachEnergyFromDiscardToBench` (Dynamotor),

## AI summary

- **Greedy step loop** with priority ordering: bench Basics → search
  Items → activate free-value abilities → evolve → fill bench → play
  Supporter → Stadium → attach Energy → defensive/offensive switch →
  attack.
- **Per-attack 1-ply lookahead minimax.** At attack-pick time, clone the
  state, apply the candidate, drive opp's full turn (greedy), drive
  our follow-up turn (greedy), score the resulting position via
  `scorePosition` (prizes ×250, HP ÷6, energy ×8, bench depth, terminal
  ±1M for win/loss). Pick the highest-scoring branch.
- **Ability scoring covers ~70 kinds** with prize-trade awareness.
  Munkidori-style moves score by KO potential; Dusknoir self-KO scores
  by prize-swing math; Teal Dance / Hydrapple ramp top the energy lane.
- **Promote selection** is opp-aware — penalizes candidates the opp can
  OHKO (extra penalty for ex / Mega), rewards candidates that can OHKO
  back.
- **Switch heuristics** — `tryDefensiveRetreat` (active will die,
  benched option survives + can attack), `tryOffensiveSwitch` (active
  can't OHKO but bench can, with prize-trade safety check).

## Mobile / iOS / offline

- **Capacitor scaffolded.** `capacitor.config.ts` configured;
  `npx cap add ios` generates the `ios/` Xcode project, which builds
  the static `dist/` into a native WebView shell.
- **PWA via vite-plugin-pwa.** `public/manifest.webmanifest` + generated
  service worker; CacheFirst for the dataset chunk and CDN card
  images, NetworkFirst for the app shell. Installable from Chrome /
  Safari.
- **IndexedDB-backed deck imports** via `idb-keyval` (~600 bytes
  gzipped). User-imported decks survive storage pressure better than
  localStorage; UI settings remain in localStorage.
- **Mobile-responsive CSS.** Stadium overlay floats on the right edge
  (matches desktop); bench scrolls horizontally inside its column with
  `overflow-x: auto` + `safe center` flex so cards center when they fit
  but stay reachable when they overflow. Hand row scrolls
  right-to-left with momentum + scroll-snap; action bar stacks
  attacks/abilities vertically so all buttons are visible.

## Known gaps (intentional MVP scope)

- Ultra Ball auto-picks the first 2 hand cards to discard (no chooser).
- Stadium activated-effect UI buttons not wired per-Stadium (passives
  all work; activated framework exists in `stadiumActivated.ts`).
- Fossil "play as 60-HP Basic" mechanic not modeled (Antique Cover /
  Jaw / Plume / Root / Sail Fossil).
- A few multi-step interactive Supporters still auto-pick (Perrin,
  Cassiopeia, Salvatore, Colress's Tenacity).
- Older `snipeOne` bench-target picker auto-targets most-damaged.
  Newer `distributeDamage` family IS interactive.
- Prize-pick UI (player always takes top prize; no wired card needs
  prize manipulation).
- Heavy Baton auto-target is primitive.

## Decks available

Built from real card data in `src/data/decks.ts`. Each is 60 cards with
a coherent line, staple trainers, and matching basic energy:

- `festival-leads` — Festival Lead twin-strike (double-attack via
  Festival Grounds).
- `arboliva` — Teal Mask Ogerpon ramp into Arboliva ex's Oil Salvo
  spread + Aroma Shot finisher (under Forest of Vitality).
- `alakazam` — Alakazam draw engine + Battle Cage bench protection.
- `lucario-ex` (display name "Mega Lucario") — Mega Lucario ex
  Aura Jab → Mega Brave under Premium Power Pro.

Card-name limits (max 4 except basic energy) are enforced by the
builder. Custom decks can be imported via `decklistParser.ts` (PTCGL
format text) and persist to IndexedDB.

- Prefer editing existing files over adding new ones.
